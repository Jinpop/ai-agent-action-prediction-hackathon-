# 4형제 전용 분류기 seed60 (완료 — ★대성공)

## 가설 (음성결과 3건이 지목한 유일 경로)
downstream 후처리(재랭커·prior보정) 전부 실패 → 4형제 구분은 **원문에서 더 나은 표현**만이 답.
14-way 모델은 용량을 14개 클래스에 분산. **4형제(read/grep/glob/list)만 4-way로 end-to-end 학습**하면
전 용량을 그 경계에 집중 → top-1 0.604 개선 노림. (top-2가 0.815이므로 정보는 원문에 있음)

## 구성
- colab_train_base2.py + `SIBLING_ONLY=1` (28,782 4형제샘플만, 4-way head, 나머지 동일)
- roberta-base, [SRC]+512, 8ep, batch16×accum3, seed 60
- 검증됨: 필터→4way→저장 파이프라인 스모크 통과

## 통합 (추론)
- 메인 앙상블(4-seed)이 4형제로 예측한 샘플에 specialist 적용
- specialist 4-way확률 ⊕ 앙상블 4형제확률 블렌드 후 재argmax (블렌드비중은 홀드아웃서 튜닝)
- 배포안전(같은 아키텍처 213MB)

## 완료 후 기입: 4형제 홀드아웃 정확도(기존 0.604 대비), 통합 후 전체 macro, LB
