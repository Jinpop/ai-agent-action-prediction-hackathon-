# 실패/기각 실험 기록 (음성 결과도 자산)

## 4형제 2단계 재분류기 — 피처 기반 (2026-07-06, 기각)
- **가설**: 앙상블 top-2 정확도 0.815인데 top-1은 0.604 → 후보 2개 중 재결정하는 2단계 분류기로 top-1 회복
- **방법**: 앙상블이 4형제로 예측한 5,792행에 대해, [앙상블 4형제 확률 4d + 고전 상태피처 17종(sig_where/has_filename/prompt_hits_open_file/kw_*...)] → HGB 4-way 재분류, 그룹CV
- **결과**: 4형제내 정확도 0.603 → **0.522 (악화)**, 전체 macro 0.7555 → 0.7222
- **결론**: 손으로 만든 상태피처+앙상블 확률로는 top-2 타이를 못 깬다. 앙상블이 이미 그 신호를 최적 사용 중. **4형제 구분 신호는 원문에 있고 더 강한 표현학습이 필요 → 피처 엔지니어링/재랭커 경로 폐기.**
- **함의**: 다음 카드는 반드시 upstream(모델 자체). end-to-end 4형제 전용 분류기 or 더 나은 백본.

## large가 base를 못 이김 (누적 증거)
- v14 large(337M) 홀드아웃 0.7129, [SRC]-large(seed50) 0.7450 — base(110M) 0.744~0.748과 동급
- 블렌드에서도: v20(+large) 0.7634 < v21(base만) 0.7701 / v23b(+large) 0.7732 < v22 0.7746
- **결론**: 단순 파라미터 수 증가는 이 과제에서 무효. 답은 표현/접근법 or 학습 방식(large 저학습 가능성은 미검증).

## 4형제 prior 보정 (2026-07-06, 기각)
- **가설**: 4형제 argmax가 체계적 편향 → 확률에 클래스별 계수만 곱해 보정
- **결과**: 공정 그룹CV 0.7560 vs 기존 0.7555 = **+0.0005 (무의미)**, fold마다 최적계수 불일치(안정적 편향 없음)
- **결론**: argmax 편향 아님. 앙상블의 4형제 불확실성은 진짜 — downstream 후처리(재랭커·prior보정) 전부 실패. **upstream 모델만이 답.**

## LLM 교사(프런티어 few-shot) 검증 (2026-07-07 새벽, 기각)
- **가설**: 프런티어 LLM이 few-shot으로 4형제를 앙상블(0.66)보다 잘 맞히면 증류 교사로 사용
- **방법**: 홀드아웃 4형제 240행(클래스별 60 층화), few-shot 16개, 서브에이전트 8병렬. 150행 회수 시점에 판정
- **결과**: LLM 0.300 vs 앙상블 0.660 (동일 행). glob 0.09, list 0.09 — 무작위(0.25) 근처
- **결론**: 라벨은 '합리적 행동'이 아니라 **시뮬레이터의 통계적 버릇** → 일반 지능으로 못 풀고 이 데이터로 학습해야만 함. LLM 증류/API 교사 경로 완전 폐기 (외부 과금 불필요해짐)
- 부산물: 앙상블 soft label 자기증류는 여전히 유효한 후보 (외부 지능 불필요)

## mDeBERTa-v3-base (2026-07-06 밤, 중단)
- 6ep 시점 0.712 — 같은 epoch의 klue/roberta(0.747)보다 명확히 낮아 8ep 완주 전 kill
- 다국어 백본이 한국어+코드 혼합 텍스트에서 한국어 특화 roberta에 밀림. klue/bert-base(s52 0.670)도 동일 패턴
- **결론: 백본은 klue/roberta-base가 로컬 최적. 백본 교체 경로 폐기**

## MAX_LEN 768 본대 (seed70, H100, 2026-07-07 새벽, 기각)
- **가설**: 4형제의 11.7%가 512에서 잘림 → 위치임베딩 선형보간 확장(514→770)으로 768 학습하면 +
- **결과**: 6ep 홀드아웃 0.7447 (5ep 피크 0.7455) — 512 시드들(0.7467~0.7482)보다 낮음
- **결론**: 잘림은 병목 아님. 보간된 위치임베딩의 미세 손실이 이득을 상쇄하는 듯. **레시피 512 유지**
- 참고: klue/roberta는 512 하드제한 — extend_positions 패치(colab_train_base2.py)로 해제 가능함은 확인
- 산출물: hybrid/holdout_probs_768s70.npy (**H100 분할 — 제3분할군!**), submits/submit_768_s70.zip

## 10ep 장기런 (seed71, H100, 2026-07-07 새벽, 기각)
- **가설**: 8ep까지 상승 추세(s47 0.7481)의 연장 — 10ep면 더 오를까
- **결과**: ep별 0.618/0.689/0.697/0.730/0.741/**0.748(6ep피크)**/0.744/0.744/0.741/0.734 — 7ep부터 하락
- **결론**: **최적 epoch = 6~7 확정.** s47의 8ep 0.7481은 시드 운. refit도 10ep로 돼 있어 이 zip은 앙상블 부적합
- H100 속도 실측: base 512 batch48 ≈ 3.8it/s, 동시 2작업 가능, 풀런(6ep+refit) 45분

## 모델 수프 (가중치 평균, 2026-07-08 낮, 기각)
- **가설**: 시드들 가중치 평균 = 슬롯당 시드 2-3개 압축 → zip 1GB 내 실효 ~10시드
- **결과**: soup(s46+47+48) 홀드아웃 **0.6379** (단독 0.745~0.748 대비 붕괴)
- **원인**: head가 시드별 랜덤 초기화 → 뉴런 순서 불일치(permutation mismatch) → 평균 파괴. 백본도 6ep 중학습으로 서로 다른 분지에 정착
- **결론**: 수프는 "동일 init에서 갈라진 가벼운 파인튠"용 — 우리 세팅과 불일치. 용량 확장 경로 폐기

## 고전 현대화 — 풀트랜스크립트 TF-IDF (2026-07-08, 기각)
- **가설**: 고전(v7)은 2세대 전 입력(build_text) 사용 중 → 트랜스포머를 +0.05 올린 풀트랜스크립트 적용시 상승
- **결과**: Blend3 0.5989 (v7 0.6089 대비 **-0.01 후퇴**). HGB 0.535/LogReg 0.573/NB 0.329
- **원인**: bag-of-words에 긴 문서는 신호 희석 — 프롬프트·최근턴 집중형(build_text)이 TF-IDF에 최적이었음
- **결론**: 고전은 v7 그대로 유지. 산출물 artifacts/classic_v8_* (참고용)
