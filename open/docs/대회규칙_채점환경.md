## 2. 채점 환경 (공식) ★중요
| 항목 | 공식 값 | 우리 파이프라인 함의 |

## 5. 코드 검증 요구사항 (공식은 한 줄뿐)
- 명시된 것: **"학습 코드는 오류 없이 실행되어야"** 함.
- 랜덤시드/재현성 세부 기준: 언급 없음 → 보수적으로 준비:
  - 학습 스크립트 전부 seed 고정돼 있음 (42/43).
  - 학습 파이프라인 실행 순서 문서화 필요: feat.py → (고전) train.py / (트랜스포머) colab_train_hybrid2.py, colab_train_base2.py → (스태킹) stack_hybrid*.py → 포장 스크립트.
  - GroupKFold가 sklearn 버전에 따라 분할이 달라지는 문제 있음 → **검증 환경 제출 시 requirements에 sklearn 버전 고정 명시**.

## 6. 우리 실측/추정 (공식 아님 — 구분 유지)
- 표시 채점시간 = 순수 스크립트 실행시간 (검증됨).
- 서버 테스트: 세션당 1 step 구조 (v9 수확 0건으로 검증), 규모 ~2.5만 행 (v12 처리시간 역산 추정).
- 홀드아웃→LB 갭: 트랜스포머 계열 -0.008 ~ -0.019 (표본 4개).
- 제출 오류(타임아웃 등)의 횟수 차감 여부: 미확인 — QnA 게시판 확인 권장.
