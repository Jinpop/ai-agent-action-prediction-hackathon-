## ⚙️ 메타규칙 (하네스 자기갱신 — 최우선)
- **★사용자가 규칙·선호·지시를 말할 때마다 즉시 이 AGENTS.md에 추가한다.** (이 메타규칙 포함) 말로만 받고 넘어가지 말 것. 규칙 확정 즉시 하네스 반영 → 메모리(feedback)도 병행.
- **★실험/작업(학습·refit·평가 등)을 돌릴 때마다 예상 완료 시각(ETA)을 함께 명시한다.** "언제 끝나냐" 되묻게 하지 말 것. 예: "refit 3변형 ~12분, 완료 ≈ HH:MM경."
- **★ETA를 말했으면 그 시각에 스스로 이어서 작업한다.** 백그라운드 작업은 **반드시 완료감시를 붙일 것**: 로컬은 `run_in_background`로 하네스가 추적하게 하거나(nohup 단독 발사 금지 — 완료 알림 안 옴), 원격(치타)은 블로킹 워처 커맨드를 `run_in_background`로 병행. 완료 알림 수신 즉시 후속 단계(수확→포장→메모) 자동 진행.
- **★역할 분리(2026-07-11 사용자 지시): Claude가 제출 zip 생성·수정·포장과 작업 완료 감시를 담당하고, Codex는 사용자가 요청할 때만 독립 검토한다.** 사용자의 명시적 요청이 없는 한 Codex는 학습/refit을 발진하거나 모델·배포 코드를 변경하거나 제출 zip/포장 스테이징을 생성·수정하지 않으며, 치타에 접속하거나 watcher·heartbeat 등 자동 완료 감시도 붙이지 않는다. 요청 시 검토 범위는 코드·주장·산출물 provenance 감사, split/index/class-order 및 logits/probability 계약 검증, metric 독립 재계산, 누수 검사, zip 목록·용량·hash·런타임·스모크 검증이다. 검증용 scratch 산출물은 만들 수 있으나 제출 산출물로 승격하지 않는다.
- **★치타 운영 승인(2026-07-11 04:29): 사용자는 당시 진행 중인 치타 run을 중단하고 새 학습으로 교체해도 된다고 승인했다.** 현재 요청은 계획 수립이므로 실제 중단·발진은 Claude가 담당하며, Codex의 요청 시 검토 역할은 유지한다. 이 승인은 당시 run 교체에 대한 운영 승인이고 향후 모든 상태변경의 포괄 승인으로 확대하지 않는다.
- **★재현성 manifest 규칙(07-12 교정10): 전체 printenv 덤프 금지(자격증명 유출 위험).** 학습 스크립트가 파싱한 **effective config JSON**(20개 플래그 전부·기본값 포함 + 코드·데이터 SHA256 + 라이브러리 버전)을 run 디렉터리에 자동 저장(`effective_config.json`, colab_train_base2.py 구현). ±0.001급 차이는 "노이즈" 표현 금지 — "학습 draw·seed·처리 상호작용 범위". matched 실험은 frozen 디렉터리에서 실행.

## ⚖️ 핵심 규칙 요약 (상세는 HANDOFF.md)
- **챔피언은 자동 최종선택**(최고 LB 자동 채택). 라디오 수동조작 없음.
- **★홀드아웃 = 제출 자격 필터일 뿐, 우열 판단자 아님.** 홀드아웃 점수로 "대박/최고"라 부르지 말 것. 밴드(챔피언홀드−0.0058) 통과 = "제출해볼 자격"일 뿐. **우열·챔피언·어느 걸 제출할지는 LB와 전략으로만 판단.** 서버LB ≈ 홀드아웃 + 0.0131 ± 0.0039(참고). 정보축은 홀드아웃으로 죽이지 말 것. 정규화/specialist류는 서버증거로 닫힘(재시도 금지).
- **슬롯: 매일 자정(KST) 10개 리셋.** 매일 소진.
- **정보축(전달됨)**: 더 긴 컨텍스트·더 많은/좋은 데이터·refit·강한 멤버. **정규화(반납)**: FGM·증류·smooth·specialist·스태킹·CA.
- **배포:** net.half() 필수, zip 1GB, T4 600초, 결합기 없는 순수 블렌드(0.6 seed + 0.4 고전).
- **제출은 사용자 직접, 제출 zip 생성·포장·완료 감시는 Claude, Codex는 사용자 요청 시 독립 검토만 수행.** **엘리스 H100(과금)은 사전허락.** epoch별 알림 금지·완료시 소요보고.
- **파일 삭제 주의:** zsh는 변수 단어분리 안 함 → whitelist 변수로 삭제 금지(전삭제 사고 이력). find/명시적 배열 사용.
